### 사이트가 해결하는 문제

- 시설명이 비슷한 공사·청주 체력단련장을 구분한다.
- 시설별 주소, 대표전화, 추가 문의처, 코스 특징을 한 화면에서 찾는다.
- 현역·군무원·가족·예비역 기본요금과 민간인 시설별 요금을 함께 본다.
- 예약 가능 기간, 확정 시점, 취소 기준과 이용횟수를 순서대로 이해한다.
- 공식 예약체계로 바로 이동하기 전에 필요한 정보를 먼저 확인한다.

## 2. 운영 원칙

1. 공개된 공군복지시설예약체계 자료를 우선 사용한다.
2. 공식 로고와 공식 기관을 연상시키는 마크는 사용하지 않는다.
3. 예약 가능 여부, 당일 운영, 휴장 여부는 고정 정보처럼 단정하지 않는다.
4. 공식 외부 링크는 사용자가 내부 정보를 읽은 뒤 이동하도록 페이지 후반에 배치한다.
5. 전화번호는 시설 문의 동선으로 제공하며 전화예약 가능 여부는 해당 시설에 문의하도록 안내한다.
6. 같은 주소·전화·예약 문구를 여러 구역에서 반복하지 않는다.
7. 확인되지 않은 시설이나 추정 정보는 추가하지 않는다.
8. 사용자가 바로 이해할 수 있는 짧은 문장과 표를 우선한다.

### 비공식 사이트 고지

본 사이트는 국방부, 대한민국 공군, 공군복지시설예약체계와 별도로 운영되는 독립 정보 안내 사이트다. 예약, 로그인, 결제, 확정 조회와 취소를 직접 처리하지 않는다.

## 6. 페이지별 콘텐츠 설계

### 6.1 홈

홈은 세 가지 핵심 검색 의도를 대표하는 페이지다.

- H1: `공군체력단련장 예약 안내`
- title: `공군체력단련장 예약 안내 | 공군복지시설예약체계 이용정보`
- 헤더 보조 문구: `공군복지시설예약체계`

구성:

1. 공군체력단련장 예약 안내 히어로
2. 시설 선택 → 자격·요금 확인 → 예약의 3단계 흐름
3. 수도권부터 시작하는 권역 버튼
4. 전국 14개 시설의 주소·전화·상세 보기 목록
5. 예약 안내, 이용요금, 처음 방문 안내 링크
6. 출처 및 검수 기준 연결

시설 검색 입력창과 히어로의 외부 공식예약 버튼은 두지 않는다.

### 6.2 시설 목록

- 전국 14개 시설을 5개 권역으로 묶는다.
- 카드에는 시설명, 지역, 주소, 대표전화, 자세히, 전화문의를 제공한다.
- 공식 외부 링크는 시설 상세 페이지 후반에서 제공한다.
- 별도 지역 비교 페이지나 지역 비교 메뉴는 만들지 않는다.

### 6.3 시설 상세 14개

각 상세 페이지는 다음 순서를 공통으로 사용한다.

1. 시설명과 권역
2. 요금 보기와 전화문의
3. 핵심 정보: 공식 명칭, 주소, 대표전화와 추가 문의처
4. 시설 특징: 개장, 홀 구성, 코스 성격 등 공개 자료 요약
5. 찾아가는 길: 공식 경로 요약과 초행자용 이동 팁
6. 코스시설: 공식 코스 설명을 바탕으로 한 대표 홀 특징
7. 현역·군무원·가족·예비역 기본요금표
8. 해당 시설 민간인 입장요금표
9. 전체 요금표 내부 링크
10. 같은 권역 시설 링크
11. 공식 예약체계와 공식 코스 상세 링크

주소와 전화번호는 핵심 정보 구역에 한 번만 모아 보여준다. 개요, 길 안내, 예약 카드에서 동일 문구를 반복하지 않는다.

### 6.4 예약 안내

페이지 제목은 `공군체력단련장 예약 방법 | 일정·확정·취소`다.

핵심 구성:

- 시설 찾기, 회원자격, 이용요금 빠른 이동
- 시설 선택 → 자격·요금 확인 → 예약 → 확정·방문 준비
- 시설명, 예약 상태, 시간, 동반자, 신분증, 주소 체크리스트
- 취소 시각과 무단 불참 경고
- 예약 기준, 회원자격, 이용 제한 내부 링크
- 전화 문의 준비 항목
- 마지막 구역의 공식 예약체계 링크

화면 제목과 메뉴에서는 `예약`을 사용한다. 공식 화면의 실제 메뉴명을 설명할 때만 `예약신청`을 사용한다.

### 6.5 예약 기준

- 예약 가능 기간
- 예약 확정 시점
- 확정 후 취소 가능 시각
- 전산장애 시 연락 방법
- 공식 예약 화면의 입력 순서
- 배정 우선순위
- 개인별 이용횟수 기준
- 무단 불참과 동반자 변경 관련 이용 제한 연결

### 6.6 요금 안내

공식 이용요금표의 `2024년 5월 1일부` 기준을 명시한다.

표 구성:

- 현역·군무원·가족·예비역 기본요금
- 9홀 평일·휴일
- 18홀 서산 평일·휴일
- 민간인 시설별 평일·민간인의 날·휴일 요금
- 정회원·준회원·민간인 카트 이용요금
- 노캐디 코스관리비
- 월 이용횟수 초과, 민간인의 날, 병역이행자 할인, 지역별 할인·할증 안내

공통 안내는 표 행마다 반복하지 않고 표 바깥에서 한 번만 설명한다.

### 6.7 회원자격

- 정회원
- 정회원대우
- 준회원
- 준회원대우
- 민간인 회원

최종 자격은 예약체계의 회원 정보와 공식 자격 기준을 따르도록 안내한다.

### 6.8 이용 제한

1개월, 3개월, 6개월, 1년 이상, 영구이용금지 또는 자격박탈 기준을 표로 정리한다. 예약 취소, 무단 불참, 동반자 변경, 보안, 복장과 현장 질서를 함께 다룬다.

### 6.9 처음 방문 안내

- 시설명 대조
- 주소 저장
- 신분증과 동반자 정보
- 복장과 장비
- 출입 시간을 고려한 조기 도착
- 예약 화면과 대표전화 저장

### 6.10 FAQ

링크 없이 답할 수 있는 질문은 문장만 제공하고, 추가 확인 페이지가 필요한 답변에만 내부 링크를 넣는다.

주요 질문:

- 공군체력단련장 예약 방법
- 사이트의 예약 처리 여부
- 전화예약 가능 여부
- 대상별·시설별 요금
- 민간인 이용
- 주소와 전화번호
- 취소와 미방문 기준
- 정보 출처

### 6.11 신뢰·정책 페이지

- 사이트 소개
- 출처 및 검수 기준
- 문의
- 개인정보처리방침
- 이용약관

출처 페이지에는 시설 목록, 전화번호, 오시는 길, 요금, 휴장일, 예약 기준, 회원자격, 이용 제한에 사용한 공식 공개 경로를 정리한다.

## 7. 콘텐츠 작성 원칙

### 사람이 쓴 문장으로 보이게 하는 기준

- 시설마다 공개된 개장 배경, 코스 형태, 대표 홀 특징을 사용한다.
- 지역 이동 팁은 실제 도로·IC·주변 검색어와 연결한다.
- `먼저 볼 것`, `이 시설을 볼 때` 같은 동일 소제목을 모든 시설에 복제하지 않는다.
- 주소와 전화번호를 설명 문장마다 다시 쓰지 않는다.
- 같은 의미의 예약 고지를 세 번 이상 반복하지 않는다.
- 길게 늘인 일반론보다 사용자가 다음 행동을 결정할 수 있는 문장을 쓴다.
- 공식 자료를 그대로 길게 복제하지 않고 읽기 쉬운 요약으로 재구성한다.

### 콘텐츠 중복 관리

- 전역 공통 정책은 전용 안내 페이지 한 곳에서 자세히 설명한다.
- 시설 상세에는 해당 시설에 필요한 핵심 내용만 넣는다.
- 요금 공통 기준은 전체 요금 페이지와 시설별 표로 역할을 나눈다.
- 외부 공식 링크 안내는 페이지 마지막 구역에 모은다.
- FAQ는 안내 페이지의 문장을 그대로 복사하지 않고 질문에 직접 답한다.

## 8. SEO 전략

### 핵심 검색어와 대표 페이지

| 검색 의도 | 대표 페이지 | 적용 위치 |
|---|---|---|
| 공군체력단련장 | 홈·시설 목록 | title, H1, 시설 목록, 내부 링크 |
| 공군체력단련장 예약 | 홈·예약 안내 | 홈 H1, 예약 안내 title, 단계형 본문 |
| 공군복지시설예약체계 | 홈 | 헤더 보조 문구, title, description, 예약 단계 |
| 시설명 + 체력단련장 | 시설 상세 | 고유 title, H1, 주소·전화·코스 내용 |
| 공군체력단련장 이용요금 | 요금 안내 | title, H1, 공식 표 기반 콘텐츠 |

키워드는 검색결과만을 위해 반복하지 않는다. 제목, H1, 요약과 본문이 같은 검색 의도를 일관되게 설명하도록 구성한다.

### 페이지 SEO

- 색인 페이지마다 고유한 title과 meta description을 사용한다.
- H1은 페이지당 한 개만 사용한다.
- 운영 도메인이 확정되면 canonical과 `og:url`을 페이지별 절대주소로 적용한다.
- Open Graph와 Twitter 메타를 title·description과 함께 생성한다.
- 시설 상세는 BreadcrumbList와 GolfCourse 정보를 사용한다.
- 홈은 WebSite와 14개 시설 ItemList 구조화 데이터를 사용한다.
- 요금·출처 등 주요 안내 페이지는 WebPage 구조화 데이터를 사용한다.
- 모든 색인 페이지는 최소 두 곳 이상의 내부 유입 링크를 갖는다.
- `sitemap.xml`과 `robots.txt`는 페이지를 추가하거나 주소가 바뀔 때 함께 갱신한다.
- noindex 호환 페이지는 검색 대표 페이지와 경쟁하지 않도록 한다.

### 검색 등록

배포 후 다음 작업을 진행한다.

1. 네이버 서치어드바이저에 운영 도메인 등록
2. 사이트 소유 확인
3. `sitemap.xml` 제출
4. 홈, 시설 목록, 예약 안내, 요금 안내 URL 검사
5. Google Search Console 등록과 사이트맵 제출
6. 색인된 title과 description이 의도대로 노출되는지 점검

## 10. 모바일 UX와 접근성

모바일 방문자를 우선한다.

- 360px 전후 화면에서 가로 넘침이 없도록 한다.
- 메뉴는 아이콘 버튼으로 열고 닫는다.
- 홈 권역 버튼은 짧은 가로 스크롤을 허용한다.
- 시설 목록과 안내 단계는 한 열로 전환한다.
- 전화번호는 `tel:` 링크로 바로 연결한다.
- 시설 상세 하단에는 요금과 전화문의 중심의 모바일 행동 영역을 제공한다.
- 표는 짧은 표는 화면 폭에 맞추고, 복잡한 표는 모바일 데이터 라벨로 행을 재구성한다.
- 금액의 숫자와 `원`은 분리되지 않게 한다.
- 긴 대상명은 자연스럽게 줄바꿈하고 열 너비를 내용량에 맞춘다.
- 본문은 `word-break: keep-all`을 기본으로 하되 긴 단어는 화면을 넘지 않게 처리한다.
- 본문 바로가기, 의미 있는 landmark, aria-label, 현재 메뉴 표시를 제공한다.
- 키보드 포커스와 버튼 대비를 유지한다.

## 11. 성능과 보안

### 성능

- 원본 CSS와 JavaScript를 수정한 뒤 배포용 최소화 파일을 함께 갱신한다.
- 히어로 이미지는 900px·1600px 두 크기로 제공한다.
- 홈의 LCP 이미지는 preload와 `fetchpriority="high"`를 사용한다.
- 외부 폰트와 무거운 UI 프레임워크를 사용하지 않는다.
- JavaScript는 메뉴와 최소 상호작용에만 사용하고 defer로 불러온다.
- 파일 내용 해시를 자산 URL에 붙여 캐시 갱신을 제어한다.
- 정적 자산은 장기 캐시, HTML은 재검증 캐시를 사용한다.
- 승인 전 빈 광고 영역은 렌더링하지 않는다.

### 보안 헤더

`_headers`와 `.htaccess`에서 다음 정책을 제공한다.

- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-Frame-Options: SAMEORIGIN`
- 카메라·마이크·위치 권한 비활성화

## 14. 배포 전 설정

### 운영 도메인 적용

현재 로컬 미리보기 주소는 검색 등록에 사용할 운영 주소가 아니다. 실제 도메인이 확정되면 다음 값을 배포본에 반영한다.

| 항목 | 적용 내용 |
|---|---|
| canonical·`og:url` | 페이지별 운영 도메인 절대주소 |
| `sitemap.xml` | 모든 `loc`을 운영 도메인 절대주소로 변경 |
| `robots.txt` | 운영 사이트맵 절대주소 추가 |
| 문의처 | 공개 가능한 운영자 이메일 또는 문의 방법 |

### 배포 점검

1. 운영 도메인의 절대주소를 HTML과 사이트맵에 적용한다.
2. canonical과 sitemap의 절대주소를 확인한다.
3. 문의 이메일과 개인정보처리방침을 대조한다.
4. HTTPS와 보안 헤더를 확인한다.
5. 모바일 360px, 390px과 데스크톱 화면을 점검한다.
6. 네이버 서치어드바이저와 Google Search Console에 등록한다.
7. PageSpeed Insights에서 LCP, CLS, INP를 측정한다.
8. 광고 삽입 후 레이아웃 이동과 본문 가림이 없는지 다시 확인한다.

## 15. 유지보수 원칙

- 변경 후 최소화 파일과 자산 버전을 갱신하고 정적 검사와 모바일 화면 검수를 실행한다.
