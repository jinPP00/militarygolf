# 육군체력단련장 안내 사이트 기획서

> 목적: 전국 육군체력단련장의 시설, 입장요금, 위치, 예약 절차와 이용 규정을 한곳에서 비교할 수 있는 정보 사이트  
> 운영 방향: 사용자가 사이트 안에서 충분히 정보를 확인한 뒤 예약할 때만 공식 예약 화면이나 시설 예약실을 이용하도록 구성  
> 운영 고지: 본 사이트는 육군이나 국군복지단이 운영하는 공식 사이트가 아니며 예약·결제 기능을 제공하지 않음  
> 현행 기준: 2026년 7월 구현 상태

---

## 1. 프로젝트 개요

| 항목 | 현행 기획 |
|---|---|
| 사이트명 | 육군체력단련장 안내 |
| 성격 | 전국 육군체력단련장 골프장 정보 안내 사이트 |
| 주요 사용자 | 시설 선택, 입장요금 비교, 예약 준비, 방문 정보를 찾는 이용자 |
| 핵심 목표 | 시설 탐색 → 요금·위치·코스 확인 → 예약 절차 확인 → 예약 수단 선택 |
| 제작 방식 | Node.js 생성기 기반 정적 HTML/CSS/JS |
| 데이터 기준 | 육군체력단련장 공식 홈페이지의 시설·예약/경기/요금 공개 화면 |
| SEO 핵심어 | 육군체력단련장, 육군체력단련장 예약 |
| 수익화 방향 | 충분한 고유 콘텐츠와 정책 페이지를 갖춘 뒤 애드센스 신청 검토 |

사이트의 우선순위는 공식 사이트로 즉시 이동시키는 것이 아니다. 홈과 시설 상세 페이지에서 시설, 요금, 위치, 코스와 이용 정보를 먼저 제공하고, 예약을 진행하는 마지막 단계에만 공식 예약 화면과 전화문의 수단을 배치한다.

## 4. 페이지별 콘텐츠 설계

### 4.1 홈

- 첫 화면에서 사이트명과 전국 12개 시설 범위를 즉시 전달
- 시설명·지역·권역·홀 수로 검색
- 수도권·충청권·영남권·호남권 필터
- 시설, 지역, 코스, 시설 안내, 소재지 날씨를 한 행에서 확인
- 시설 상세, 예약 안내, 이용 규정으로 연결
- 회원 구분, 합계 요금, 예약 신청과 확정의 차이를 실제 판단 순서로 안내
- `육군체력단련장` 검색 의도를 담당
- 공식 예약 링크나 전화번호는 첫 시설표에 노출하지 않음

### 4.2 시설 안내

- 전체 12곳, 4개 권역, 18홀 2곳, 9홀 10곳 현황 표시
- 시설명, 지역, 권역, 코스, 일반 이용자 대표 요금 비교표
- 데스크톱에서는 표 전체를 한 화면 폭에 배치
- 모바일에서만 필요한 경우 가로 스크롤 허용
- 권역별 시설 디렉터리 제공

### 4.3 시설 상세

각 페이지는 같은 정보 구조를 사용하되 설명 문장은 지역과 시설 특성에 맞게 다르게 작성한다.

1. 시설명, 지역, 코스, 대표 요금 요약
2. 시설 개요
3. 상세 주소와 내비게이션 검색명
4. 카카오맵 버튼
5. 일반 이용자 입장요금표
6. 회원·할인 구분, 요일, 그린피, 카트료, 합계
7. 코스 선택 방식과 희망 차수
8. 주요 편의시설
9. 이동·시설 구분·예약 준비 메모
10. 페이지 마지막의 공식 예약 화면과 예약실 전화문의
11. 이전·다음 시설 내부 링크
12. 시설별 예약 전 질문 2개
13. 공식 자료 확인 기준일과 정보 정정 경로

요금 표기는 각 숫자에 `원`을 붙이며 별도 단위 문구를 반복하지 않는다. 할인 증빙, 보수료처럼 요금 구조에 영향을 주는 항목은 시설별 표와 주석에 반영한다.

### 4.4 예약 안내

`육군체력단련장 예약` 검색 의도를 담당하며 예약 관련 표현을 하나로 통일한다.

- 시설 선택 → 예약 접수 → 확정 확인의 3단계
- 정회원·정회원대우·준회원·준회원대우·비회원 구분
- 일반적인 접수 기간: 경기일 15일 전부터 9일 전 24:00까지
- 예약 편성: 경기일 8일 전
- 확정 확인: 경기일 7일 전부터
- 정상 취소 기준: 경기일 3일 전 13:00까지
- 예약 전 준비 항목과 출발 전 준비 항목
- 예약신청확인과 예약확정확인의 차이 및 이동 경로
- 경기일이 20일인 경우의 날짜 계산 예시
- 시설별 확인이 필요한 경우에만 전화문의 안내
- 공식 예약 버튼은 페이지 마지막에 배치

### 4.5 이용 규정

- 정상 취소와 지연 취소의 시점별 제한
- 무단불참과 예약 양도 등 이용 제한 사례
- 경기 30분 전 도착 권장
- 티오프 10분 전 출발점 도착
- 4인 1팀 기준
- 모자, 긴 바지, 깃 있는 상의, 골프화 안내
- 현장 표식과 시설별 로컬룰 우선

### 4.6 FAQ

- 답만으로 충분한 질문에는 링크를 넣지 않음
- 상세 기준을 함께 봐야 하는 답에만 예약 안내, 시설 안내, 이용 규정 링크 제공
- FAQ 구조화 데이터의 질문과 화면 질문을 동일하게 유지
- 핵심 질문: 육군체력단련장 예약 방법, 전화문의 번호, 접수 기간, 취소 제한, 요금 기준, 방문 준비

### 4.7 소개·정책 페이지

- 자료의 출처와 직접 작성한 콘텐츠의 범위 구분
- 공개 자료 확인 기준일과 시설별 자료 기준 표시
- 공식 서비스와 민간 정보 안내 사이트의 역할 구분
- 회원가입, 결제, 예약 접수, 자격 판정을 처리하지 않음을 명시
- 문의 이메일은 운영 빌드의 `CONTACT_EMAIL` 값으로 표시
- 예약 문의와 사이트 정보 정정 문의를 전용 페이지에서 구분
- 요금·예약·지도·날씨의 책임 범위를 별도 정보 이용 안내에 정리
- 광고 도입 전 쿠키와 동의 관리 내용을 실제 운영 방식에 맞게 갱신

## 6. SEO 설계

### 검색 의도 분담

| 페이지 | 핵심 검색 의도 |
|---|---|
| 홈 | 육군체력단련장 |
| 예약 안내 | 육군체력단련장 예약 |
| 시설 안내 | 육군체력단련장 시설, 전국 시설, 입장요금 |
| 시설 상세 | 시설명 + 입장요금·코스·오시는 길 |
| 이용 규정 | 육군체력단련장 취소·노쇼·복장 |
| FAQ | 예약·요금·방문 준비 질문 |

### 기술 SEO

- 모든 색인 페이지에 고유한 title, meta description, H1
- canonical과 Open Graph URL
- 1200×630 PNG Open Graph 대표 이미지와 large image 카드
- `WebSite`, `WebPage`, `ItemList`, `BreadcrumbList`, `FAQPage`, `GolfCourse` JSON-LD
- 내부 링크로 홈, 시설 목록, 시설 상세, 예약 안내, 이용 규정 연결
- 운영 빌드에 22개 색인 URL을 담은 `sitemap.xml`
- 로컬 빌드는 `noindex, nofollow`와 robots 전체 차단
- `NODE_ENV=production`과 `SITE_URL`이 모두 있을 때만 색인 허용
- 운영 빌드는 `SITE_URL`과 `CONTACT_EMAIL`이 없으면 중단
- 배포 후 Google Search Console과 네이버 서치어드바이저에 사이트맵 제출

키워드는 같은 문장이나 제목에 반복해서 넣지 않는다. 제목은 검색 의도를 분명히 전달하되 사람이 읽었을 때 자연스러운 길이로 유지한다.

## 7. 콘텐츠 및 애드센스 원칙

- 주소와 전화번호만 모은 얇은 페이지로 만들지 않음
- 시설별 지역 맥락, 이동 준비, 시설 구분과 코스 정보를 고유하게 작성
- 공식 화면의 표를 그대로 복제하는 데 그치지 않고 비교·확인 순서를 제공
- 검증되지 않은 운영시간, 휴장일, 예약 가능 여부를 임의로 작성하지 않음
- 공식 기관으로 오인시키는 로고, 명칭, 예약 기능을 사용하지 않음
- 정책 페이지와 사이트 소개를 항상 접근 가능하게 유지
- 광고는 승인 후 콘텐츠를 가리거나 예약 버튼처럼 보이지 않는 위치에 배치
- 애드센스 승인은 콘텐츠 품질 외에도 도메인, 운영 기간, 정책 준수 등 외부 요인의 영향을 받으므로 승인 자체를 보장하지 않음

## 8. 성능 및 유지보수

- 외부 UI 프레임워크와 플러그인 없이 공통 CSS와 최소 JavaScript 사용
- `build.js`의 시설 데이터로 12개 상세 페이지를 일괄 생성
- CSS, JS, SVG에 버전 쿼리를 붙여 캐시 갱신
- 미리보기 서버는 ETag, Last-Modified, Cache-Control 사용
- 긴 하단 섹션은 지원 브라우저에서 `content-visibility` 적용
- SVG 파비콘, 사이트 마크, 날씨, 지도 아이콘 사용
- 29KB PNG 공유 대표 이미지를 빌드 시 복사
- 시설 검색과 권역 필터만 JavaScript로 처리
- 데이터 수정은 생성기의 시설 정보, 요금표, 예약 프로필을 고친 뒤 전체 재빌드

---

## 9. 운영 및 갱신 절차

1. 공식 홈페이지의 시설별 주소, 전화번호, 요금, 예약 기준 변경 확인
2. `build.js`의 해당 데이터 수정
3. `node build.js`로 로컬 페이지 생성
4. 홈, 시설 안내, 예약 안내, 이용 규정과 수정 시설을 데스크톱·모바일에서 확인
5. 운영 빌드에 실제 `SITE_URL`, `CONTACT_EMAIL`, `OUTPUT_DIR` 설정
6. canonical, robots, sitemap, 구조화 데이터와 내부 링크 검사
7. 호스팅에서 `_redirects`의 초보자 안내 → 예약 안내 301 적용 확인
8. 없는 주소가 `404.html`과 HTTP 404 상태를 반환하는지 확인
9. 정적 호스팅에 운영 출력물만 배포
10. Google Search Console과 네이버 서치어드바이저에서 색인 상태 확인

### 운영 빌드 예시

```powershell
$env:NODE_ENV="production"
$env:SITE_URL="https://실제-도메인"
$env:CONTACT_EMAIL="실제-문의-이메일"
$env:OUTPUT_DIR="dist"
node build.js
```
